# Вход и создание

## Задание

#  Shell: Вход и создание 

HTTP API 5 сообщений

Начать испытание 

В этом упражнении вам нужно отправить POST-запрос для создания нового пользователя на сервере. Чтобы это сделать вам также потребуется предоставить авторизационный Bearer-токен для аутентификации.

## solution

  * Используя _curl_ выполните post-запрос к _localhost_ на порт 8080 по пути _/login_ , чтобы авторизоваться в приложении, в качестве логина и пароля используйте `admin`
  * Используя _curl_ выполните post-запрос к _localhost_ на порт 8080 по пути _/users_ , чтобы отправить данные нового пользователя в формате JSON
  * Тело запроса для создания нового пользователя должно содержать поля с такими данными: 
    * `username: guest`
    * `email: guest@example.com`
    * `password: guest123`
  * Если запрос успешный, сервер вернет статус _201 (Created)_ и данные о созданном пользователе в формате JSON.
  * Получившуюся команду для создания нового пользователя запишите в файл _solution_  




### Подсказки

  * Убедитесь, что вы отправляете правильный заголовок `Content-Type` с типом `application/json`
  * Кроме того, убедитесь, что вы предоставляете корректный авторизационный Bearer-токен в заголовке запроса для аутентификации. Токен может быть предоставлен в заголовке `Authorization`
