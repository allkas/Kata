Начать испытание 

В современном мире музыкальные сервисы стали частью нашей повседневной жизни, предоставляя доступ к миллионам песен на любой вкус. Часто бывает так, что мы хотим найти песню по названию, но не помним название целиком. В этом упражнении мы создадим простой музыкальный сервис, который позволяет искать композиции по части имени

## scr/main/java/io/hexlet/model/Song.java

Создайте класс `Song`, который будет представлять собой композицию в музыкальном сервисе. У композиции есть уникальный идентификатор, название и исполнитель. Добавьте в класс необходимые свойства и методы по своему усмотрению

## scr/main/java/io/hexlet/dao/SongDAO.java

Создайте класс `SongDAO`, который предназначен для работы с таблицей композиций _songs_. Создайте в классе метод, который будет получать из таблицы все записи, в названии которых содержится переданная подстрока. Структуру таблицы вы можете посмотреть в файле _init.sql_

## scr/main/java/io/hexlet/Catalog.java

В классе `MusicService` создайте метод `getSongByTitle()`. Метод принимает один строковый параметр — условие поиска. Метод должен вернуть список песен `List<Song>`, в названии которых содержится переданная подстрока

Если совпадений не найдено, метод должен вернуть пустой список

URL базы данных, в которой содержатся записи о музыкальных композициях нашего сервиса: _jdbc:h2:./hexlet_

### Подсказки

  * Загляните в файл _init.sql_ и в тесты для полноты картины

## Код

**DataInitializer.java**
```java
package io.hexlet.component;

import java.nio.file.Files;
import java.nio.file.Paths;
import java.sql.Connection;

public class DataInitializer {
    public static void init(Connection connection) throws Exception {

        var path = Paths.get("init.sql");
        var sql = Files.readString(path);

        try (var statement = connection.createStatement()) {
            statement.execute(sql);
        }
    }
}
```
