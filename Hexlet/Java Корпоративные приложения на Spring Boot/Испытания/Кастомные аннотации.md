Начать испытание 

Часто в разработке возникает задача проверить, какой метод был вызван и с какими аргументами. Для этого используется логирование. Чтобы не писать каждый раз логирование вручную, создадим свою аннотацию, которой можно будет отметить класс. При вызове методов аннотированного класса в консоль будет записываться лог. Этот лог содержит имя метода и аргументы, с которым он был вызван.

## src/main/java/io/hexlet/app/Inspect.java

Создайте аннотацию `Inspect`, которая отмечает класс. Аннотация будет указывать, что при вызове методов отмеченного класса в консоль будет выводиться лог с именем метода и аргументами, с которыми он был вызван. Аннотация должна быть доступна в рантайме. Аннотация должна принимать один аргумент `level`, который будет настраивать уровень логирования. Условимся, что параметр может иметь только два значения: "info" и "debug". По умолчанию он равен "debug". Пример использования аннотации:
    
    
      @Inspect(level = "info")
      class SomeClass {
          // Методы класса
      }
    

## src/main/java/io/hexlet/app/calculator

Для проверки работы аннотации создан простой интерфейс `Calculator` и класс `CalculatorImpl`, который реализует этот интерфейс. Изучите файлы в директории. Обратите внимание, что класс `CalculatorImpl` отмечен созданной вами аннотацией `@Inspect`.

## src/main/java/io/hexlet/app/CustomBeanPostProcessor.java

Просто создать аннотацию недостаточно, нужно подсказать фреймворку, как с ней работать. Для этого будем использовать интерфейс `BeanPostProcessor`, который позволяет нам управлять жизненным циклом бина. Для работы нам понадобится рефлексия и прокси – специальный объект, который позволяет оборачивать исходный и управлять вызовом его методов.

Создайте класс `CustomBeanPostProcessor`, который реализует интерфейс `BeanPostProcessor`. Не забудьте отметить класс нужной аннотацией, чтобы Spring создал его экземпляр при старте приложения.

В классе создайте два метода: `postProcessBeforeInitialization()` и `postProcessAfterInitialization()`. Сигнатуру методов можно посмотреть в документации.

Метод `postProcessBeforeInitialization()` выполняется перед инициализацией каждого бина. В этом методе нужно проверить, отмечен ли бин аннотацией `@Inspect`. Если отмечен, нужно запомнить этот бин и нужный уровень логирования.

Метод `postProcessAfterInitialization()` выполняется после инициализации каждого бина. В этом методе нужно обернуть отмеченные аннотацией `@Inspect` бины в прокси. Вызов методов аннотированного класса должен выводить в консоль лог с нужным уровнем. Лог содержит имя метода и параметры, с которым метод был вызван. Пример сообщения: _Was called method: sum() with parameters: [3, 4]_
    
    
      // Класс
      @Inspect(level = "info")
      public class CalculatorImpl implements Calculator {
          // Методы класса
      }
    
      // Где-то в контроллере
      System.out.println(calc.sum(5, 4));
    

Вывод лога в консоли:
    
    
      # Выводится лог с уровнем INFO
      Was called method: sum() with parameters: [5, 4]
      9
    

Запустите приложение и отправьте несколько _POST_ запросов на адреса _/sum_ и _/mult_. Параметры для сложения и умножения передаются в теле запроса в виде JSON. Например, _POST_ запрос на _/sum_ с телом
    
    
    {
        "firstOperand": 5, 
        "secondOperand": 7
    
    }
    

должен вернуть ответ c результатом
    
    
    {
        "result": 12
    }
    

Убедитесь, что в консоль выводится лог с нужным уровнем. 

Вот так может выглядеть лог в консоли:
    
    
    2024-05-15T11:17:25.571+03:00  INFO 44553 --- [0.0-8080-exec-3] io.hexlet.app.CustomBeanPostProcessor    : Was called method: sum() with arguments: [5, 7]
    

Попробуйте изменить уровень логирования в аннотации и в конфигурации приложения.

## Подсказки

  * Интерфейс BeanPostProcessor – позволяет модифицировать бины в процессе создания
  * Класс Class – содержит полезные методы для работы с рефлексией
  * Класс Proxy – позволяет создать прокси, который оборачивает исходный класс и позволяет настраивать вызовы методов
  * Для работы с прокси изучите документацию и код в директории Example
  * Чтобы отправлять POST запросы приложению, можете использовать любую утилиту для работы с HTTP, например Postman. Url приложения скопируйте из адресной строки браузера на странице Web-доступ в упражнении
  * Изменения, которые вы сделаете в исходном коде после старта приложения, не будут подтягиваться автоматически. Чтобы учесть сделанные изменения, остановите приложение и вновь запустите его командой `make start`
  * Чтобы получить значение параметра, переданного в аннотацию, нужно вызвать у аннотации одноименный метод. Его вызов вернет значение параметра.
  * Чтобы увидеть в консоли логи уровня DEBUG, измените уровень логгирования на DEBUG в файле _application.yml_

**CalculationController.java**
```java
package io.hexlet.app.controller;

import io.hexlet.app.calculator.Calculator;
import io.hexlet.app.dto.CalculationDTO;
import io.hexlet.app.dto.ResultDTO;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class CalculationController {

    @Autowired
    private Calculator calculator;

    @PostMapping("/sum")
    public ResultDTO getSum(@RequestBody CalculationDTO data) {
        int result = calculator.sum(data.getFirstOperand(), data.getSecondOperand());
        return new ResultDTO(result);
    }

    @PostMapping("/mult")
    public ResultDTO getMult(@RequestBody CalculationDTO data) {
        int result = calculator.mult(data.getFirstOperand(), data.getSecondOperand());
        return new ResultDTO(result);
    }
}
```
