Игральная кость — это шестигранный кубик, который бросается несколько раз. Гистограмма — это графическое представление данных в виде столбцов или колонок.

## scr/main/java/io/hexlet/App.java

В классе `App` реализуйте публичный статический метод `play()`, который выводит на экран горизонтальную гистограмму. Метод принимает на вход два параметра:

  * Количество бросков кубика (целое число `int`)
  * Функцию, которая реализует функциональный интерфейс `Supplier`, имитирующий бросок игральной кости. Вызов этой функции генерирует целое число от 1 до 6, что соответствует одной из граней игральной кости. Саму эту функцию реализовывать не нужно



Гистограмма содержит строки, каждой из которых соответствует грань игральной кости и количество выпадений этой грани. Результаты отображаются графически с помощью символов `#` и в виде числового значения за исключением случаев, когда количество равно нулю:
    
    
    App.play(100, rollDie);
    // => 1|####################### 23
    //    2|################## 18
    //    3|############# 13
    //    4|#################### 20
    //    5|############ 12
    //    6|############## 14
    
    App.play(13, rollDie);
    // => 1|
    //    2|## 2
    //    3|# 1
    //    4|## 2
    //    5|#### 4
    //    6|#### 4
    

### Подсказки

  * Гистограмма
  * Функциональный интерфейс Supplier
  * Методы generate() и limit() из класса `Stream` помогут создать стрим определенной длины и заполнить его значениями, сгенерированными переданной функцией `Supplier`
