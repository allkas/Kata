# ArrayList vs LinkedList
## 1. Устройство, вставка, поиск
**ArrayList** — динамический массив.
- Хранит элементы подряд в памяти.
- Вставка в конец: O(1) амортизированно.
- Вставка в середину: O(n) (сдвиг).
- Поиск по индексу: O(1).
**LinkedList** — двусвязный список.
- Элементы хранятся в узлах со ссылками на соседей.
- Вставка в начало/конец: O(1).
- Вставка в середину: O(n) (нужно дойти до позиции).
- Поиск по индексу: O(n).
## 2. Сложность (Big O)
| Операция | ArrayList | LinkedList |
|----------|-----------|------------|
| add(E e) | O(1) | O(1) |
| add(index, e) | O(n) | O(n) |
| get(index) | O(1) | O(n) |
| remove(index) | O(n) | O(n) |
| remove(Object) | O(n) | O(n) |
## 3. Когда что использовать
**ArrayList** — когда:
- Частый доступ по индексу.
- Вставка только в конец.
- Меньше памяти (нет ссылок на соседей).
**LinkedList** — когда:
- Частая вставка/удаление в начале/конце.
- Реализация очереди/стека (хотя лучше ArrayDeque).
## 4. Как расширяется ArrayList
- Начальная capacity = 10 (по умолчанию).
- При заполнении: `newCapacity = oldCapacity + (oldCapacity >> 1)` (в 1.5 раза).
- Копирование всех элементов в новый массив O(n).
