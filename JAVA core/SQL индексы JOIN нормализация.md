# SQL индексы JOIN нормализация
## 1. Виды индексов
- **B-tree** — по умолчанию, для равенства и диапазонов.
- **Hash** — только для равенства.
- **GiST, GIN** — для полнотекстового поиска, JSONB.
Индекс ускоряет поиск (O(log n) вместо O(n)).
## 2. Когда индекс вреден
- Много вставок/обновлений (индекс перестраивается).
- Мало данных (план может игнорировать индекс).
- Селективность < 5-10%.
## 3. JOIN'ы
- **INNER JOIN** — только совпадающие.
- **LEFT JOIN** — все из левой + совпадения правой.
- **RIGHT JOIN** — все из правой.
- **FULL JOIN** — все из обеих.
## 4. WHERE vs HAVING
- **WHERE** — фильтр строк ДО группировки.
- **HAVING** — фильтр групп ПОСЛЕ группировки.
## 5. Нормализация
1NF — атомарность, 2NF — нет частичных зависимостей, 3NF — нет транзитивных.
## 6. Оптимизация запросов
- **EXPLAIN ANALYZE** — план выполнения.
- Искать Seq Scan, узкие места.
- Добавлять индексы, переписывать запросы.
