# Контракт equals и hashCode
## 1. Что такое equals() и hashCode()? Контракт, зачем переопределять вместе
**equals()** — сравнивает объекты по содержимому.  
**hashCode()** — возвращает целочисленный хеш-код объекта.
**Контракт:**
1. Если `equals()` вернул true, `hashCode()` должен быть одинаковым.
2. Если `hashCode()` разный, `equals()` должен вернуть false.
3. Обратное НЕ требуется: одинаковый hashCode может быть у разных объектов (коллизия).
**Зачем переопределять вместе:** Коллекции (HashMap, HashSet) используют hashCode для поиска корзины, а equals для сравнения внутри корзины. Без переопределения оба работают по адресу в памяти (), что ломает логику.
## 2. Если hashCode() всегда возвращает одно и то же число — что будет?
Все объекты попадут в одну корзину (бакет). HashMap превратится в связанный список с поиском O(n) вместо O(1).
## 3. Связь с коллекциями (HashMap, HashSet)
- **HashMap** — сначала hashCode() ищет бакет, потом equals() сравнивает ключи.
- **HashSet** — использует HashMap внутри с фиктивным значением.
## 4. Что, если изменить поля у объекта-ключа после помещения в HashMap?
hashCode() изменится, но объект останется в старом бакете. Найти его станет невозможно (утечка памяти). Ключи должны быть immutable.
