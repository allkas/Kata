# Многопоточность основы
## 1. volatile
**volatile** — гарантирует:
- Видимость изменений для всех потоков (не кэшируется в регистрах).
- Запрещает переупорядочивание (happens-before).
**Не гарантирует:** атомарность (i++ всё равно не атомарно).
## 2. volatile vs synchronized
| volatile            | synchronized            |
| ------------------- | ----------------------- |
| Только видимость    | Видимость + атомарность |
| Не блокирует потоки | Блокирует               |
| Для одного поля     | Для блока/метода        |
| Легче               | Тяжелее                 |
## 3. Deadlock, livelock, race condition
- **Race condition** — гонка потоков за ресурс, непредсказуемый результат.
- **Deadlock** — потоки ждут друг друга вечно.
- **Livelock** — потоки постоянно меняют состояние, но прогресса нет.
## 4. Атомарные типы
AtomicInteger, AtomicLong — CAS-операции без synchronized, быстрее при конкурентности.
## 5. Runnable vs Callable
| Runnable | Callable |
|----------|----------|
| run() | call() |
| Не возвращает результат | Возвращает T |
| Не бросает checked | Бросает Exception |
## 6. ThreadLocal
Хранит копию переменной для каждого потока (например, Connection, SecurityContext).
