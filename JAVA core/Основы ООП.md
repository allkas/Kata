# Основы ООП
## 1. Что такое ООП? Принципы
**ООП** — методология программирования, основанная на объектах.
**Принципы:**
1. **Инкапсуляция** — скрытие внутренней реализации, доступ через методы.
2. **Наследование** — создание новых классов на основе существующих.
3. **Полиморфизм** — способность объекта принимать разные формы.
4. **Абстракция** — выделение существенных характеристик без деталей.
## 2. Интерфейс vs абстрактный класс
| Интерфейс | Абстрактный класс |
|-----------|-------------------|
| Только public методы (до Java 8) | Может иметь поля и конструкторы |
| Множественное наследование | Одиночное наследование |
| Все поля static final | Обычные поля |
| Java 8+: default/static методы | Может быть частичная реализация |
## 3. Можно ли переопределить статический метод?
Нет — статические методы не переопределяются, а **скрываются** (hide). Вызов зависит от типа ссылки, а не объекта.
## 4. Смысл инкапсуляции
Защита данных от некорректного использования. Пример: поля private, геттеры/сеттеры с валидацией.
## 5. Полиморфизм: перегрузка vs переопределение
- **Перегрузка (overload)** — методы с одинаковым именем, разными параметрами (compile-time).
- **Переопределение (override)** — изменение реализации метода родителя (runtime).
