# Рекурсия и мемоизация

## 1. Проблема — зачем это существует?

Многие задачи имеют рекурсивную природу: дерево — это узел + два поддерева, факториал — это n * factorial(n-1), обход директорий — обход + рекурсивный обход поддиректорий. Писать итеративный аналог сложнее и менее выразительно. Но наивная рекурсия пересчитывает одни и те же подзадачи — мемоизация кэширует результаты и превращает экспоненциальный алгоритм в полиномиальный.

## 2. Аналогия

**Рекурсия** — как матрёшка: чтобы открыть большую, открываешь меньшую, ещё меньшую... пока не дойдёшь до самой маленькой (базовый случай), которая уже не открывается.

**Мемоизация** — как шпаргалка. Первый раз решил задачу — записал ответ. Второй раз та же задача — достал шпаргалку, не думаешь заново.

## 3. Как работает

### Структура рекурсии

Любая рекурсия состоит из двух частей:
1. **Базовый случай** — самая простая задача, решается напрямую (условие выхода)
2. **Шаг рекурсии** — сведение задачи к меньшей подзадаче

```java
// Классический пример — факториал:
int factorial(int n) {
    if (n == 0) return 1;           // базовый случай
    return n * factorial(n - 1);   // шаг рекурсии
}

// Числа Фибоначчи (наивная рекурсия — O(2^n)):
int fib(int n) {
    if (n <= 1) return n;           // базовый случай
    return fib(n - 1) + fib(n - 2); // два рекурсивных вызова
}
```

**Как работает стек вызовов:**

```
factorial(4)
  → 4 * factorial(3)
         → 3 * factorial(2)
                → 2 * factorial(1)
                       → 1 * factorial(0)
                              → 1 (базовый случай)
                       = 1 * 1 = 1
                = 2 * 1 = 2
         = 3 * 2 = 6
  = 4 * 6 = 24
```

### Рекурсия vs Итерация

| | Рекурсия | Итерация |
|---|---|---|
| **Читаемость** | Лучше для иерархических задач | Лучше для последовательных |
| **Производительность** | Хуже (overhead на вызовы методов) | Лучше |
| **Память** | O(n) стек вызовов → StackOverflowError | O(1) |
| **Отладка** | Сложнее | Проще |
| **Применение** | Деревья, графы, divide & conquer | Массивы, списки, простые циклы |

```java
// Итеративный factorial (лучше по производительности):
int factorialIterative(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}
```

**Хвостовая рекурсия (tail recursion):** если рекурсивный вызов — последнее действие в методе. Компилятор может оптимизировать в итерацию (Java, к сожалению, не делает tail call optimization).

### Мемоизация

Кэшируем результаты вычислений — при повторном вызове с теми же аргументами возвращаем кэшированный ответ.

**Проблема наивной рекурсии Fibonacci:**

```
fib(5)
├── fib(4)
│   ├── fib(3)
│   │   ├── fib(2) ✓
│   │   └── fib(1) ✓
│   └── fib(2) ← ПОВТОР!
└── fib(3) ← ПОВТОР!
    ├── fib(2) ← ПОВТОР!
    └── fib(1) ← ПОВТОР!
```

**O(2^n)** вызовов для наивного Fibonacci!

**С мемоизацией — O(n):**

```java
// Вариант 1: HashMap как кэш
Map<Integer, Long> memo = new HashMap<>();

long fibMemo(int n) {
    if (n <= 1) return n;
    if (memo.containsKey(n)) return memo.get(n);  // кэш-хит!
    long result = fibMemo(n - 1) + fibMemo(n - 2);
    memo.put(n, result);
    return result;
}

// Вариант 2: массив
long[] dp = new long[n + 1];
Arrays.fill(dp, -1);

long fibDp(int n) {
    if (n <= 1) return n;
    if (dp[n] != -1) return dp[n];  // кэш-хит!
    dp[n] = fibDp(n - 1) + fibDp(n - 2);
    return dp[n];
}

// Вариант 3: итерация (лучший — O(n) time, O(1) space)
long fibIterative(int n) {
    if (n <= 1) return n;
    long prev = 0, curr = 1;
    for (int i = 2; i <= n; i++) {
        long next = prev + curr;
        prev = curr;
        curr = next;
    }
    return curr;
}
```

### Динамическое программирование (DP)

Мемоизация = "top-down DP" (рекурсия + кэш). "Bottom-up DP" — строим таблицу результатов снизу вверх (итерация).

```java
// Задача: монеты — минимальное количество монет для суммы amount
// coins = [1, 5, 10, 25], amount = 36

// Рекурсия с мемоизацией (top-down):
int coinChangeMemo(int[] coins, int amount, int[] memo) {
    if (amount == 0) return 0;
    if (amount < 0) return -1;
    if (memo[amount] != 0) return memo[amount];

    int min = Integer.MAX_VALUE;
    for (int coin : coins) {
        int sub = coinChangeMemo(coins, amount - coin, memo);
        if (sub >= 0) min = Math.min(min, sub + 1);
    }
    memo[amount] = (min == Integer.MAX_VALUE) ? -1 : min;
    return memo[amount];
}

// Итерация (bottom-up DP):
int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1);  // "бесконечность"
    dp[0] = 0;
    for (int i = 1; i <= amount; i++) {
        for (int coin : coins) {
            if (coin <= i) {
                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
            }
        }
    }
    return dp[amount] > amount ? -1 : dp[amount];
}
```

### Полезные рекурсивные паттерны

```java
// Обход дерева (DFS):
void dfs(TreeNode root) {
    if (root == null) return;  // базовый случай
    process(root.val);
    dfs(root.left);            // рекурсивный шаг
    dfs(root.right);
}

// Обход директорий:
void listFiles(File dir) {
    if (!dir.isDirectory()) {  // базовый случай — файл
        System.out.println(dir.getName());
        return;
    }
    for (File f : dir.listFiles()) {
        listFiles(f);          // рекурсивный шаг
    }
}

// Бинарный поиск (рекурсивный):
int binarySearch(int[] arr, int target, int low, int high) {
    if (low > high) return -1;  // базовый случай — не найден
    int mid = low + (high - low) / 2;
    if (arr[mid] == target) return mid;
    if (arr[mid] > target) return binarySearch(arr, target, low, mid - 1);
    return binarySearch(arr, target, mid + 1, high);
}
```

## 4. Глубже — что важно знать

**StackOverflowError:** рекурсия без базового случая или с очень глубоким стеком → переполнение стека вызовов. Для больших n (> ~10000) лучше итерация.

**`@Cacheable` в Spring** — мемоизация на уровне фреймворка: результаты методов кэшируются в Redis/Caffeine, повторные вызовы с теми же аргументами не выполняют метод.

**Жадный алгоритм vs DP:** жадный алгоритм на каждом шаге делает локально наилучший выбор, надеясь на глобальный оптимум. DP рассматривает все возможности. Пример: алгоритм Дейкстры (кратчайший путь) — жадный, но доказуемо оптимальный. Задача о рюкзаке — требует DP.

**Tail Call Optimization:** хвостовая рекурсия теоретически оптимизируется компилятором в итерацию (O(1) стек). Java JVM не делает TCO — нужна ручная конвертация в итерацию.

```java
// Хвостовая рекурсия — последнее действие = рекурсивный вызов:
int factTail(int n, int accumulator) {
    if (n == 0) return accumulator;
    return factTail(n - 1, n * accumulator); // tail call — Java не оптимизирует
}
```

## 6. Ответ на собесе (2 минуты)

> "Рекурсия — функция, вызывающая сама себя. Состоит из базового случая (условие выхода) и рекурсивного шага (уменьшение задачи).
>
> **Рекурсия vs Итерация:** рекурсия — выразительнее для иерархических задач (деревья, разделяй-и-властвуй). Итерация — быстрее (нет overhead на вызовы методов) и безопаснее (нет StackOverflowError).
>
> **Мемоизация** — кэширование результатов рекурсии. Наивный Fibonacci — O(2^n), с мемоизацией — O(n). Реализую через HashMap или массив. Мемоизация = top-down DP.
>
> **Bottom-up DP** — строим таблицу от меньших подзадач к большим итеративно. Часто эффективнее по памяти.
>
> **Жадный алгоритм** — локально оптимальный выбор на каждом шаге. Не всегда глобально оптимален — нужно доказывать. Дейкстра, жадное покрытие — жадные. Задача о рюкзаке — нет, нужен DP."

| | Рекурсия | Мемоизация (top-down DP) | Bottom-up DP |
|---|---|---|---|
| Структура | Функция вызывает себя | Рекурсия + HashMap/массив | Таблица, итерация |
| Сложность | Зависит от задачи | Оптимальная | Оптимальная |
| Память стека | O(n) — риск SOE | O(n) | O(1) возможно |
| Читаемость | Высокая | Средняя | Ниже |

| Fibonacci | Сложность | Память |
|-----------|-----------|--------|
| Наивная рекурсия | O(2^n) | O(n) стек |
| С мемоизацией | O(n) | O(n) кэш + O(n) стек |
| Итерация | O(n) | O(1) |
