## 1. Как устроена HashMap (бакеты, коллизии, loadFactor)
- **Массив бакетов (корзин)** — начальный размер 16.
- **Коллизия** — разные ключи с одинаковым hashCode попадают в один бакет.
- **Load factor** (0.75) — когда заполнено 75% бакетов, размер удваивается (rehashing).
- В Java 8+ если в бакете > 8 элементов, список превращается в красно-черное дерево (ускорение поиска).
## 2. Коллизии в HashMap, когда превращается в дерево
- Порог **TREEIFY_THRESHOLD = 8** — список -> дерево.
- Порог **UNTREEIFY_THRESHOLD = 6** — дерево -> список.
## 3. Можно ли класть null в HashMap / HashSet / TreeMap
- **HashMap**: да, один null-ключ.
- **HashSet**: да, один null (через HashMap).
- **TreeMap**: нет, требует сравнения (NullPointerException).
## 4. Разница HashMap / TreeMap / LinkedHashMap
| Тип | Порядок | Сложность | Особенность |
|-----|---------|-----------|--------------|
| HashMap | Нет | O(1) | Самый быстрый |
| LinkedHashMap | Вставки | O(1) | Двусвязный список |
| TreeMap | Сортированный | O(log n) | Красно-черное дерево |
## 5. Сложность операций в HashMap
- Вставка/поиск/удаление — **O(1)** в среднем, **O(n)** в худшем (коллизии).
- При дереве — O(log n).
## 6. HashSet, LinkedHashSet, TreeSet
- **HashSet** — неупорядоченный, O(1).
- **LinkedHashSet** — порядок вставки, O(1).
- **TreeSet** — сортированный, O(log n), требует Comparable/Comparator.
