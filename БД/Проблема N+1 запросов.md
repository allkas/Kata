# Проблема N+1 запросов
## 1. Проблема N+1
При загрузке сущности с коллекцией (OneToMany) Hibernate делает:
- 1 запрос на родителя
- N запросов на каждую дочернюю сущность
## 2. Решения
- **JOIN FETCH** в JPQL.
- **@EntityGraph**.
- **Batch fetching**.
- **DTO проекции**.
## 3. LazyInitializationException
Когда LAZY коллекция загружается вне сессии/транзакции.
## 4. LAZY vs EAGER
- **EAGER** — загружает сразу (может быть N+1 или гигантский JOIN).
- **LAZY** — загружает при первом обращении (но нужна открытая сессия).
## 5. JOIN FETCH vs обычный JOIN
- **JOIN** — только для WHERE/условий, данные не загружаются.
- **JOIN FETCH** — загружает данные в persistence context.
