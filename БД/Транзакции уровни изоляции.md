# Транзакции уровни изоляции
## 1. ACID
- **A**tomicity — атомарность
- **C**onsistency — согласованность
- **I**solation — изоляция
- **D**urability — долговечность
## 2. Уровни изоляции
| Уровень | Грязное чтение | Неповторяемое чтение | Фантомы |
|---------|----------------|----------------------|---------|
| Read Uncommitted | ✅ | ✅ | ✅ |
| Read Committed | ❌ | ✅ | ✅ |
| Repeatable Read | ❌ | ❌ | ✅ |
| Serializable | ❌ | ❌ | ❌ |
## 3. @Transactional в Spring
Транзакция начинается при входе в метод, заканчивается при выходе. Прокси через AOP.
## 4. rollbackFor
По умолчанию откат только на RuntimeException и Error. Для checked нужно указывать:
```java
@Transactional(rollbackFor = Exception.class)
```
## 5. Propagation

- **REQUIRED** — использует существующую или создаёт новую.
    
- **REQUIRES_NEW** — всегда новая, приостанавливает текущую.
    
- **SUPPORTS** — если есть, использует; если нет, без транзакции.
    
- **MANDATORY** — обязана быть существующая.
    
- **NEVER** — не должно быть транзакции.
    
- **NESTED** — вложенная (savepoint).
