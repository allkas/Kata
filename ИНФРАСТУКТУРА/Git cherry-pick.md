# 🍒 Git cherry-pick
## 🧠 Ментальная модель
**Cherry-pick** = взять **один конкретный коммит** из другой ветки и перенести в текущую.

Ветка A: A1 ← A2 ← A3  
↑  
берём A2

Ветка B: B1 ← A2'

text

## 📋 Когда использовать
| Сценарий | Пример |
|----------|--------|
| Срочный фикс в релизную ветку | `git cherry-pick abc123` |
| Ошибочный коммит не в ту ветку | `git cherry-pick` в правильную |
| Выборочный перенос изменений | Без слияния всей ветки |
## ⚡ Пример
```bash
git checkout main
git cherry-pick abc123def   # перенести коммит в main
git cherry-pick -n abc123   # перенести без коммита (интерактивно)
```
## ⚠️ Важно

- Новый коммит получает **новый хэш**
    
- Могут быть **конфликты** — разрешаются как при merge
