# 🔀 Git merge vs rebase
## 🧠 Ментальная модель
| | **Merge** | **Rebase** |
|--|----------|-----------|
| Образ | Склеиваем две верёвки узлом | Перематываем одну ветку на другую |
| История | Сохраняет реальную хронологию | Создаёт линейную историю |
| Коммиты | Создаёт merge commit | Перезаписывает коммиты |
## 📊 Сравнение
| Характеристика | Merge | Rebase |
|----------------|-------|--------|
| Линейная история | ❌ | ✅ |
| Сохраняет контекст | ✅ | ❌ |
| Риск для public веток | Низкий | Высокий |
| Сложность разрешения конфликтов | Один раз | Каждый коммит |
## ⚡ Когда что использовать
| Сценарий | Merge | Rebase |
|----------|-------|--------|
| Ветка `main` (публичная) | ✅ | ❌ |
| Локальная ветка перед PR | ❌ | ✅ |
| Слив фичи в dev | ✅ | ⚠️ |
> [!warning] Золотое правило
> **Никогда не делайте rebase публичных веток, на которых уже кто-то работает.**
