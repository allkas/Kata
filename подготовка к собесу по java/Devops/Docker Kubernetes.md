# Docker / Kubernetes
## Docker vs VM
- **VM** — своя ОС, гипервизор, гигабайты.
- **Docker** — общее ядро, изоляция, мегабайты.
## Dockerfile
```dockerfile
FROM openjdk:17
COPY target/app.jar app.jar
CMD ["java", "-jar", "app.jar"]
```
## Kubernetes

- **Pod** — один или несколько контейнеров.
    
- **Deployment** — управление репликами.
    
- **Service** — стабильный доступ к Pod'ам.
    

## 6. Ответ на собесе (2 минуты)

> "Docker — это инструмент контейнеризации. В отличие от виртуальной машины, контейнер не несёт свою ОС — он разделяет ядро хоста и изолируется через namespace и cgroups. Результат: запуск за секунды вместо минут, образ занимает мегабайты вместо гигабайт.
>
> Dockerfile описывает образ слоями. Каждая инструкция — новый слой, кэшируемый Docker. Поэтому зависимости (COPY pom.xml + mvn dependency:resolve) нужно копировать до исходного кода — если код поменялся, слой с зависимостями не пересобирается.
>
> Kubernetes — это оркестратор контейнеров. Базовые объекты: Pod — минимальная единица (один или несколько контейнеров), Deployment — управляет репликами подов с rolling update, Service — стабильный DNS и балансировка нагрузки к подам, которые могут перезапускаться с новым IP."
