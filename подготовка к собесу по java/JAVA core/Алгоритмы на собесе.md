# Алгоритмические задачи на собесе

## 1. Проблема — зачем это существует?

Live coding — обязательная часть Java-собеседования. Задача не проверить знание алгоритмов наизусть, а увидеть: как кандидат думает, выбирает структуры данных и оценивает сложность. 90% задач решаются через HashMap, два указателя или рекурсию с мемоизацией — достаточно знать 5–7 паттернов решения.

## 2. Аналогия

Задачи на собесе — как шахматные этюды. Позиция специально составлена, чтобы проверить конкретный приём. Знаешь приёмы (HashMap для подсчёта частот, два указателя для отсортированных данных, мемоизация для рекурсии) — решаешь большинство задач за 5 минут.

## 3. Как работает

### Задача 1: Найти первый неповторяющийся элемент в массиве (freq 6)

**Постановка:** `[1, 2, 3, 2, 1, 4]` → вернуть `3` (первый элемент, встречающийся ровно один раз).

**Ключевой инсайт:** `LinkedHashMap` сохраняет порядок вставки — нам нужна и частота, и порядок.

```java
// O(n) time, O(n) space
public static Integer firstNonRepeating(int[] arr) {
    // LinkedHashMap: порядок вставки + подсчёт частот
    Map<Integer, Integer> freq = new LinkedHashMap<>();

    for (int x : arr) {
        freq.merge(x, 1, Integer::sum); // freq[x]++ (или создать с 1)
    }

    for (Map.Entry<Integer, Integer> entry : freq.entrySet()) {
        if (entry.getValue() == 1) {
            return entry.getKey(); // первый с count=1 — это и есть ответ
        }
    }

    return null; // все элементы повторяются
}

// Тест
System.out.println(firstNonRepeating(new int[]{1, 2, 3, 2, 1, 4})); // 3
System.out.println(firstNonRepeating(new int[]{1, 1, 2, 2}));       // null
```

**Почему не `HashMap`?** `HashMap` не гарантирует порядок итерации — мы можем вернуть не первый уникальный, а произвольный.

### Задача 2: Факториал / Число Фибоначчи, оценка сложности (freq 5)

#### Факториал

```java
// Рекурсия — O(n) time, O(n) space (стек вызовов)
public static long factorialRec(int n) {
    if (n <= 1) return 1;              // base case
    return n * factorialRec(n - 1);   // recursive case
}

// Итерация — O(n) time, O(1) space (предпочтительно в продакшене)
public static long factorialIter(int n) {
    long result = 1;
    for (int i = 2; i <= n; i++) result *= i;
    return result;
}
```

#### Фибоначчи — три подхода с разной сложностью

```java
// ❌ Наивная рекурсия — O(2^n) time! fib(40) ~ 1 миллиард операций
public static int fibNaive(int n) {
    if (n <= 1) return n;
    return fibNaive(n - 1) + fibNaive(n - 2); // fib(n-2) вычисляется заново
}

// ✅ Мемоизация (Top-Down DP) — O(n) time, O(n) space
public static long fibMemo(int n, Map<Integer, Long> memo) {
    if (n <= 1) return n;
    if (memo.containsKey(n)) return memo.get(n); // кэш-хит
    long result = fibMemo(n - 1, memo) + fibMemo(n - 2, memo);
    memo.put(n, result);
    return result;
}

// ✅✅ Итерация (Bottom-Up DP) — O(n) time, O(1) space — оптимально
public static long fibIter(int n) {
    if (n <= 1) return n;
    long prev = 0, curr = 1;
    for (int i = 2; i <= n; i++) {
        long next = prev + curr;
        prev = curr;
        curr = next;
    }
    return curr;
}
```

**Таблица сложностей — что спросят на собесе:**

| Алгоритм | Time | Space | Примечание |
|----------|------|-------|-----------|
| Факториал рекурсия | O(n) | O(n) | стек вызовов глубиной n |
| Факториал итерация | O(n) | O(1) | предпочтительно |
| Фибоначчи наивная рекурсия | **O(2^n)** | O(n) | катастрофа, не использовать |
| Фибоначчи с мемоизацией | O(n) | O(n) | HashMap как кэш |
| Фибоначчи итерация | O(n) | **O(1)** | оптимально |

### Задача 3: Вывести элементы, повторяющиеся N раз (freq 4)

**Постановка:** `[1, 2, 2, 3, 3, 3, 4]`, N=2 → `[2, 3]` (все элементы с частотой ≥ N).

```java
// O(n) time, O(n) space
public static List<Integer> elementsRepeatingNTimes(int[] arr, int n) {
    Map<Integer, Integer> freq = new HashMap<>();

    for (int x : arr) {
        freq.merge(x, 1, Integer::sum);
    }

    return freq.entrySet().stream()
        .filter(e -> e.getValue() >= n)
        .map(Map.Entry::getKey)
        .collect(Collectors.toList());
}

// Тест
System.out.println(elementsRepeatingNTimes(new int[]{1, 2, 2, 3, 3, 3, 4}, 2)); // [2, 3]
System.out.println(elementsRepeatingNTimes(new int[]{1, 2, 2, 3, 3, 3, 4}, 3)); // [3]
```

### Задача 4: Является ли строка палиндромом (freq 3)

**Постановка:** `"racecar"` → `true`, `"hello"` → `false`.

```java
// Вариант 1 — StringBuilder.reverse() (лаконично, O(n) space)
public static boolean isPalindrome(String s) {
    String cleaned = s.toLowerCase().replaceAll("[^a-z0-9]", ""); // убираем пробелы/знаки
    return cleaned.equals(new StringBuilder(cleaned).reverse().toString());
}

// Вариант 2 — два указателя (лучше для собеса: показывает понимание алгоритма)
// O(n) time, O(1) space
public static boolean isPalindromeTwoPointers(String s) {
    int left = 0, right = s.length() - 1;
    while (left < right) {
        if (s.charAt(left) != s.charAt(right)) return false;
        left++;
        right--;
    }
    return true;
}

// Тест
System.out.println(isPalindromeTwoPointers("racecar")); // true
System.out.println(isPalindromeTwoPointers("hello"));   // false
System.out.println(isPalindromeTwoPointers("a"));       // true (edge case)
```

**Паттерн "два указателя":** один с начала, другой с конца, движутся навстречу. Работает для отсортированных массивов и строк. O(n) time, O(1) space — лучше реверса строки.

### Задача 5: Отсортировать квадраты чисел массива (freq 3)

**Постановка:** отсортированный массив `[-4, -1, 0, 3, 10]` → `[0, 1, 9, 16, 100]`.

```java
// Наивный подход: возвести в квадрат + Arrays.sort() — O(n log n)
public static int[] sortedSquaresNaive(int[] arr) {
    int[] result = Arrays.stream(arr).map(x -> x * x).toArray();
    Arrays.sort(result);
    return result;
}

// Оптимальный — два указателя — O(n)
// Ключевой инсайт: наибольшие квадраты всегда на краях отсортированного массива
// (большие по модулю числа — с концов). Заполняем результат с конца.
public static int[] sortedSquaresOptimal(int[] arr) {
    int n = arr.length;
    int[] result = new int[n];
    int left = 0, right = n - 1;

    for (int i = n - 1; i >= 0; i--) {
        int leftSq  = arr[left]  * arr[left];
        int rightSq = arr[right] * arr[right];
        if (leftSq > rightSq) {
            result[i] = leftSq;
            left++;
        } else {
            result[i] = rightSq;
            right--;
        }
    }
    return result;
}

// Тест
System.out.println(Arrays.toString(sortedSquaresOptimal(new int[]{-4, -1, 0, 3, 10})));
// [0, 1, 9, 16, 100]
```

## 4. Глубже — что важно знать

**Паттерны решения задач (учить как приёмы):**

| Паттерн | Когда применять | Пример |
|---------|-----------------|--------|
| **HashMap для частот** | Подсчёт вхождений, поиск дублей | Задачи 1, 3 |
| **LinkedHashMap** | Нужна частота + сохранение порядка | Задача 1 |
| **Два указателя** | Отсортированный массив/строка | Задачи 4, 5 |
| **Рекурсия + мемоизация** | Перекрывающиеся подзадачи | Фибоначчи |
| **Bottom-Up DP** | Оптимизация рекурсии до O(1) space | Фибоначчи |
| **Stream API + filter** | Трансформация и фильтрация коллекций | Задача 3 |

**Нотация Big O — что надо уметь называть:**

| Нотация | Название | Пример |
|---------|----------|--------|
| O(1) | Константное | Получить элемент по индексу в ArrayList |
| O(log n) | Логарифмическое | Бинарный поиск, TreeMap |
| O(n) | Линейное | Один проход по массиву |
| O(n log n) | Линейно-логарифмическое | Arrays.sort(), merge sort |
| O(n²) | Квадратичное | Вложенные циклы |
| O(2^n) | Экспоненциальное | Наивная рекурсия Фибоначчи ❌ |

**Как вести себя на live coding:**
1. Проговорить задачу вслух — убедиться в понимании
2. Спросить edge cases: пустой массив? null? отрицательные числа?
3. Назвать структуру данных и подход *до* кодирования
4. Написать код с комментариями ключевых мест
5. Назвать Time + Space сложность после написания
6. Запустить мысленно на тестовых данных

**Edge cases, которые забывают:**
- Пустой массив или пустая строка → сразу вернуть дефолт
- Массив из одного элемента
- Все элементы одинаковые
- Отрицательные числа (критично для задачи 5!)
- `null` на входе → NullPointerException

## 6. Ответ на собесе (2 минуты)

На live coding задачах я работаю по схеме: уточняю условие и edge cases, называю структуру данных и алгоритм, потом пишу код.

Для большинства задач на поиск дублей или первого уникального — HashMap с подсчётом частот, O(n) time и space. Если нужен порядок — LinkedHashMap, он хранит порядок вставки в отличие от HashMap.

Для задач на отсортированном массиве — паттерн "два указателя". Квадраты отсортированного массива: наибольшие квадраты на краях, двигаем указатели навстречу и заполняем результат с конца. O(n) без дополнительной сортировки.

Для рекурсивных задач — факториал/Фибоначчи — сразу называю сложность. Наивная рекурсия Фибоначчи O(2^n) — это катастрофа, fib(40) займёт секунды. Решение: мемоизация через HashMap — O(n) time, O(n) space. Оптимально — итерация: O(n) time, O(1) space, только два числа prev и curr.
