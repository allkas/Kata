# Бинарное дерево и красно-чёрное дерево

## 1. Проблема — зачем это существует?

Массив даёт O(1) доступ по индексу, но O(n) поиск. LinkedList даёт O(1) вставку, но O(n) поиск. Бинарное дерево поиска (BST) балансирует: O(log n) на поиск, вставку, удаление. Но несбалансированное BST вырождается в список — O(n) в худшем. Красно-чёрное дерево — самобалансирующееся BST: гарантирует O(log n) всегда через инварианты "цвета" узлов.

## 2. Аналогия

**BST** — как игра "угадай число от 1 до 100". Каждый вопрос "больше или меньше 50?" делит пространство пополам — O(log n). Но если числа идут по порядку (1, 2, 3, ...), дерево вырождается в список — как если бы всегда угадывал с конца.

**Красно-чёрное дерево** — как тот же поиск, но с модератором, который после каждого ответа перебалансирует список вопросов. Никогда не даст вырождения.

## 3. Как работает

### Бинарное дерево поиска (BST)

**Инвариант:** для любого узла X:
- Все узлы **левого** поддерева < X
- Все узлы **правого** поддерева ≥ X

```
       8
      / \
     3   10
    / \    \
   1   6    14
      / \   /
     4   7 13
```

**Основные операции:**

```java
class TreeNode {
    int val;
    TreeNode left, right;
    TreeNode(int val) { this.val = val; }
}

// Поиск — O(log n) средний, O(n) худший (вырожденное дерево)
TreeNode search(TreeNode root, int target) {
    if (root == null || root.val == target) return root;
    return target < root.val
        ? search(root.left, target)
        : search(root.right, target);
}

// Вставка — O(log n) средний
TreeNode insert(TreeNode root, int val) {
    if (root == null) return new TreeNode(val);
    if (val < root.val) root.left = insert(root.left, val);
    else root.right = insert(root.right, val);
    return root;
}

// In-order обход (Left → Root → Right) — даёт отсортированный порядок
void inOrder(TreeNode node) {
    if (node == null) return;
    inOrder(node.left);
    System.out.print(node.val + " "); // посещаем узел
    inOrder(node.right);
}
```

**Три способа обхода:**

| Обход | Порядок | Применение |
|-------|---------|-----------|
| **In-order** (LNR) | left → node → right | Получить отсортированный массив |
| **Pre-order** (NLR) | node → left → right | Копирование/сериализация дерева |
| **Post-order** (LRN) | left → right → node | Удаление дерева, вычисление выражений |

```java
// Поиск в ширину (BFS) — уровень за уровнем:
void bfs(TreeNode root) {
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    while (!queue.isEmpty()) {
        TreeNode node = queue.poll();
        System.out.print(node.val + " ");
        if (node.left != null) queue.offer(node.left);
        if (node.right != null) queue.offer(node.right);
    }
}
```

**Проблема несбалансированности:**

```
Вставка 1, 2, 3, 4, 5 в порядке:
1
 \
  2
   \
    3      → вырождение в список! Поиск O(n), не O(log n)
     \
      4
       \
        5
```

### Красно-чёрное дерево (Red-Black Tree)

Самобалансирующееся BST. После вставки/удаления автоматически перебалансируется через **ротации** и **перекрашивание**, гарантируя высоту O(log n).

**5 инвариантов (правил):**
1. Каждый узел **красный** или **чёрный**
2. Корень — **чёрный**
3. Все листья (NIL-узлы) — **чёрные**
4. Оба потомка **красного** узла — **чёрные** (нет двух красных подряд)
5. Любой путь от узла до листа содержит **одинаковое число чёрных** узлов

```
        13 (B)
       /       \
    8 (R)     17 (R)
   /   \      /    \
 1(B)  11(B) 15(B) 25(B)
  \           \    /
  6(R)       16(R) 22(R)
```

**Следствие правил:** путь от корня до самого дальнего листа не более чем **вдвое длиннее** пути до самого ближнего → высота ≤ 2·log(n+1) → все операции O(log n).

**Где используется в Java:**
- `TreeMap` — Red-Black Tree, ключи отсортированы
- `TreeSet` — Red-Black Tree, элементы отсортированы
- `HashMap` — при >= 8 элементов в бакете цепочка → Red-Black Tree (Java 8+)

```java
// TreeMap — отсортированная карта (Red-Black Tree):
TreeMap<String, Integer> map = new TreeMap<>();
map.put("banana", 2);
map.put("apple", 1);
map.put("cherry", 3);

map.firstKey();              // "apple"
map.lastKey();               // "cherry"
map.headMap("banana");       // {"apple": 1}
map.tailMap("banana");       // {"banana": 2, "cherry": 3}
map.subMap("apple", "cherry"); // {"apple": 1, "banana": 2}

// TreeSet — отсортированное множество:
TreeSet<Integer> set = new TreeSet<>(List.of(5, 1, 3, 2, 4));
set.first();    // 1
set.last();     // 5
set.floor(3);   // 3 (наибольший ≤ 3)
set.ceiling(3); // 3 (наименьший ≥ 3)
set.lower(3);   // 2 (строго меньше 3)
set.higher(3);  // 4 (строго больше 3)
```

## 4. Глубже — что важно знать

**BST vs Binary Heap:**

| | BST (TreeMap) | Binary Heap (PriorityQueue) |
|---|---|---|
| Поиск произвольного | O(log n) | O(n) |
| Min/Max | O(log n) | O(1) |
| Вставка | O(log n) | O(log n) |
| Обход in-order | O(n) отсортированный | O(n log n) |
| Применение | Словарь с порядком | Priority Queue, Top-K |

**Балансировка:** красно-чёрное дерево при вставке может нарушить инварианты — Spring исправляет через:
1. **Перекрашивание** — изменить цвет узла
2. **Левая ротация** — правый потомок становится новым корнем поддерева
3. **Правая ротация** — левый потомок становится новым корнем поддерева

**Почему не AVL-дерево?** AVL строже сбалансировано (разница высот ≤ 1) — более быстрый поиск, но медленнее вставка/удаление (больше ротаций). Red-Black — более практичен для частых изменений.

**В HashMap:** если в одном бакете >= 8 узлов — цепочка превращается в Red-Black Tree → O(log n) вместо O(n) при коллизиях. При уменьшении до 6 — обратно в список.

## 6. Ответ на собесе (2 минуты)

> "Бинарное дерево поиска — иерархическая структура, где для каждого узла: левое поддерево < узел, правое ≥ узел. Поиск, вставка, удаление — O(log n) в среднем. Но если данные идут по порядку — дерево вырождается в список, O(n).
>
> **Красно-чёрное дерево** — самобалансирующееся BST. 5 инвариантов через цвета (красный/чёрный): корень чёрный, нет двух красных подряд, одинаковое число чёрных узлов на любом пути до листа. Это гарантирует: высота ≤ 2·log(n) → все операции O(log n) гарантированно.
>
> В Java: `TreeMap` и `TreeSet` — это Red-Black Tree, дают отсортированный обход. В `HashMap` при 8+ коллизиях бакет-список превращается в Red-Black Tree.
>
> **Три способа обхода BST:** in-order (LNR) — даёт отсортированный результат, pre-order (NLR) — для копирования, post-order (LRN) — для удаления."

## Шпаргалка

| | BST | Red-Black Tree |
|---|---|---|
| Поиск | O(log n) avg / O(n) worst | O(log n) всегда |
| Вставка | O(log n) avg / O(n) worst | O(log n) всегда |
| Сбалансировано | Нет | Да (автоматически) |
| В Java | — | TreeMap, TreeSet, HashMap (бакеты) |

| Обход | Порядок | Результат |
|-------|---------|-----------|
| In-order (LNR) | left → node → right | Отсортированный |
| Pre-order (NLR) | node → left → right | Сериализация |
| Post-order (LRN) | left → right → node | Удаление |
| BFS | Уровень за уровнем | Поиск по уровням |

| Правило Red-Black | Суть |
|-------------------|------|
| 1 | Узел красный или чёрный |
| 2 | Корень — чёрный |
| 3 | NIL-листья — чёрные |
| 4 | У красного — оба потомка чёрные |
| 5 | Путь до листа: одинаковое число чёрных |
