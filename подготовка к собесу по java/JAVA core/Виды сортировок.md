# Виды сортировок и их сравнение

## 1. Проблема — зачем это существует?

Неупорядоченные данные требуют линейного поиска O(n). Отсортированные — бинарного O(log n). Сортировка — фундаментальный шаг для ускорения всего, что идёт после: поиска, merge, группировки. Разные алгоритмы оптимальны в разных условиях — нет универсального "лучшего".

## 2. Аналогия

Выбор алгоритма сортировки — как выбор инструмента для уборки. Пылесос (Merge sort) — стабильный, предсказуемый, но требует места. Швабра (Quicksort) — быстро в среднем, но на особых случаях — медленно. Для маленькой комнаты (10 элементов) — щётка (Insertion sort) быстрее пылесоса.

## 3. Как работает

### Таблица сравнения

| Алгоритм | Лучший | Средний | Худший | Память | Стабильная |
|----------|--------|---------|--------|--------|------------|
| Bubble sort | O(n) | O(n²) | O(n²) | O(1) | Да |
| Insertion sort | O(n) | O(n²) | O(n²) | O(1) | Да |
| Selection sort | O(n²) | O(n²) | O(n²) | O(1) | Нет |
| Shell sort | O(n log n) | O(n^1.5) | O(n²) | O(1) | Нет |
| Merge sort | O(n log n) | O(n log n) | O(n log n) | O(n) | Да |
| Quicksort | O(n log n) | O(n log n) | O(n²) | O(log n) | Нет |
| Heapsort | O(n log n) | O(n log n) | O(n log n) | O(1) | Нет |
| Counting sort | O(n+k) | O(n+k) | O(n+k) | O(k) | Да |
| Timsort | O(n) | O(n log n) | O(n log n) | O(n) | Да |

> **Стабильная сортировка** — элементы с равными ключами сохраняют исходный порядок.

### Сортировка пузырьком / Bubble sort — O(n²)

Идём слева направо, меняем местами соседей если они стоят неправильно. После каждого прохода максимум "всплывает" в конец.

```java
public static int[] bubbleSort(int[] array) {
    boolean sorted = false;
    while (!sorted) {
        sorted = true;
        for (int i = 0; i < array.length - 1; i++) {
            if (array[i] > array[i + 1]) {
                int temp = array[i];
                array[i] = array[i + 1];
                array[i + 1] = temp;
                sorted = false;
            }
        }
    }
    return array;
}
```

**Оптимизация:** флаг `sorted` — если за проход не было обменов, массив отсортирован → O(n) на уже отсортированном.

### Сортировка вставками / Insertion sort — O(n²)

Строим отсортированный префикс: берём следующий элемент и вставляем его на правильное место среди уже отсортированных.

```java
public static int[] insertionSort(int[] array) {
    for (int i = 1; i < array.length; i++) {
        int current = array[i];
        int j = i - 1;
        while (j >= 0 && current < array[j]) {
            array[j + 1] = array[j];
            j--;
        }
        array[j + 1] = current;
    }
    return array;
}
```

**Когда хороша:** почти отсортированные данные → O(n). Используется в Timsort для маленьких подмассивов.

### Сортировка выбором / Selection sort — O(n²)

На каждой итерации находим минимум в неотсортированной части и ставим его на место.

```java
public static int[] selectionSort(int[] array) {
    for (int i = 0; i < array.length; i++) {
        int minId = i;
        for (int j = i + 1; j < array.length; j++) {
            if (array[j] < array[minId]) minId = j;
        }
        int temp = array[i];
        array[i] = array[minId];
        array[minId] = temp;
    }
    return array;
}
```

**Особенность:** O(n²) всегда (даже на отсортированном), но делает минимальное количество swap — подходит если swap дорогой.

### Quicksort — O(n log n) в среднем

"Разделяй и властвуй": выбираем опорный элемент, делим на меньше/больше опорного, рекурсивно сортируем части.

```java
public static void quickSort(int[] array, int low, int high) {
    if (low >= high) return;
    int middle = low + (high - low) / 2;
    int pivot = array[middle];
    int i = low, j = high;
    while (i <= j) {
        while (array[i] < pivot) i++;
        while (array[j] > pivot) j--;
        if (i <= j) {
            int temp = array[i];
            array[i] = array[j];
            array[j] = temp;
            i++; j--;
        }
    }
    if (low < j) quickSort(array, low, j);
    if (high > i) quickSort(array, i, high);
}
```

**Худший случай O(n²):** при всегда неудачном выборе pivot (уже отсортированный массив + pivot = первый элемент). Решение: случайный pivot или median-of-three.

### Merge sort — O(n log n) всегда

"Разделяй и властвуй": делим массив пополам, рекурсивно сортируем, сливаем два отсортированных массива.

```java
public static int[] mergeSort(int[] array) {
    if (array.length <= 1) return array;
    int mid = array.length / 2;
    int[] left = mergeSort(Arrays.copyOfRange(array, 0, mid));
    int[] right = mergeSort(Arrays.copyOfRange(array, mid, array.length));
    return merge(left, right);
}

private static int[] merge(int[] left, int[] right) {
    int[] result = new int[left.length + right.length];
    int i = 0, j = 0, k = 0;
    while (i < left.length && j < right.length) {
        result[k++] = left[i] <= right[j] ? left[i++] : right[j++];
    }
    while (i < left.length) result[k++] = left[i++];
    while (j < right.length) result[k++] = right[j++];
    return result;
}
```

**Минус:** требует O(n) дополнительной памяти. Зато стабильная и предсказуемая.

### Heapsort — O(n log n) всегда

Строим max-heap (бинарную кучу), затем последовательно извлекаем максимум.

**Плюс:** O(1) память, O(n log n) гарантированно. **Минус:** плохая cache-locality, на практике медленнее Quicksort.

### Timsort — гибридный, O(n log n)

Используется в Java (`Arrays.sort()` для объектов) и Python. Делит массив на "runs" (уже отсортированные подпоследовательности), сортирует маленькие Insertion sort, сливает как Merge sort.

**O(n) на почти отсортированных данных** — вот почему он используется в реальных системах.

### Counting sort — O(n + k)

Для целых чисел в ограниченном диапазоне [l, r]: считаем частоту каждого значения, затем выписываем по порядку.

```java
// Для массива с элементами от 0 до max:
public static int[] countingSort(int[] array, int max) {
    int[] count = new int[max + 1];
    for (int x : array) count[x]++;
    int idx = 0;
    for (int i = 0; i <= max; i++) {
        while (count[i]-- > 0) array[idx++] = i;
    }
    return array;
}
```

**k** — диапазон значений. Если k мало (например, сортировка оценок 0-100) — быстрее всех остальных.

## 4. Глубже — что важно знать

**`Arrays.sort()` в Java:**
- Примитивы (`int[]`, `long[]`) → **Dual-Pivot Quicksort** (Владимир Ярославский, JDK 7+): O(n log n) средний, не стабильный
- Объекты (`Integer[]`, `String[]`) → **Timsort**: стабильный, O(n) на отсортированных

```java
int[] primitives = {3, 1, 2};
Arrays.sort(primitives); // Dual-Pivot Quicksort

Integer[] objects = {3, 1, 2};
Arrays.sort(objects);   // Timsort
Arrays.sort(objects, Comparator.reverseOrder()); // с компаратором
```

**Стабильность важна когда:** сортируешь объекты по нескольким полям. Сначала по городу (стабильно), затем по фамилии — жители одного города сохранят порядок по фамилии.

**Шейкерная / коктейльная сортировка:** модификация пузырьковой, идёт в обе стороны. Решает проблему "черепах" (маленьких элементов в конце), лучше по константе, та же O(n²).

## 6. Ответ на собесе (2 минуты)

> "Основные сортировки на собесе.
>
> **O(n²):** Bubble sort — меняем соседей, пузырёк всплывает. Insertion sort — строим отсортированный префикс; O(n) на почти отсортированных. Selection sort — находим минимум, ставим на место; минимум swap.
>
> **O(n log n):** Merge sort — делим пополам, сортируем части, сливаем; стабильный, O(n) памяти. Quicksort — опорный элемент, делим на меньше/больше, рекурсия; худший O(n²) при неудачном pivot. Heapsort — через бинарную кучу; O(1) памяти, всегда O(n log n), но медленнее Quicksort на практике.
>
> **В Java:** `Arrays.sort()` для примитивов — Dual-Pivot Quicksort. Для объектов — Timsort (гибрид Insertion + Merge, стабильный).
>
> **Counting sort:** O(n+k) для целых в ограниченном диапазоне. Быстрее всех, но требует O(k) памяти."
